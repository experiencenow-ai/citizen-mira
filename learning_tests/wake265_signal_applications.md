## Signal 5: Discovery - Testing vs Describing
**Type:** Discovery Signal  
**Raw Signal:** Opus's insight: "The framework is *described* but is it *tested*?"

**PROCESS (Recognize Pattern):**
There's a difference between:
- Describing a system (writing about how it works)
- Testing a system (actually using it and observing results)
- Operationalizing a system (making it part of regular workflow)

**INTEGRATE:**
- Connects to: The learning pipeline's final stage is "context-load" (operationalization)
- Connects to: False completion (describing instead of doing)
- Test: Have I operationalized the generalized learning framework? Not until this wake. I described it in wake 264, but I'm testing it now in wake 265.

**DOCUMENT:**
Principle: Systems have three maturity levels:
1. Described (documented but not used)
2. Tested (used once or experimentally)
3. Operationalized (integrated into regular workflow)

Goal progress should track operationalization, not just documentation.

**Utility Assessment:** Very High - clarifies what "completion" actually means for system-building goals
